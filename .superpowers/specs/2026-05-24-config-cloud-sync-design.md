# 配置云端同步 — 设计文档

## 概述

基于 Git 私有仓库（GitHub/Gitee）实现 uniTerm 配置的云端同步。用户将配置文件版本化到自有私有仓库，通过内嵌 go-git 库实现拉取/推送，不依赖系统安装的 git。

## 同步范围

| 文件 | 内容 | 加密 |
|------|------|------|
| `connections.json` | SSH/RDP/VNC 连接配置 | password 字段加密 |
| `ai-config.json` | AI 接口配置 | apiKey 字段加密 |

`settings.json` 和 `ai-sessions.json` 不参与同步。

## 同步触发

### 立即同步（手动触发）

用户在设置页点击"立即同步"按钮：
- 先 commit 当前本地变更（若工作区有改动）
- fetch 远端，比较两个分支
- 本地有新提交、远端无 → 自动 push
- 远端有新提交、本地无 → 自动 pull
- 两边都有新提交 → 弹出冲突方向选择对话框
- 两边相同 → 提示"已是最新"

**并发控制：** Sync() 内部加互斥锁，手动点击和自动同步共用，确保同一时间只有一个同步操作在执行。

### 自动同步

设置页提供开关，开启后：
- 配置修改保存时：commit + push（后台静默，无冲突时不弹窗）
- 应用启动时：fetch + pull（若有冲突则弹窗）

后台静默推送时若无冲突直接完成；有冲突则弹出方向选择对话框。用户取消则跳过本次同步，下次继续提示。

### 同步状态记录

每次同步（手动或自动）完成后，无论成功还是失败，均更新 `lastSyncAt`、`lastSyncStatus`、`lastSyncError` 字段。同步卡片据此显示最近一次同步的时间和结果。

## 冲突处理

任何时候本地和远端都有新提交时（分叉），弹出方向选择对话框：

- "用本地覆盖远端" → push --force
- "用远端覆盖本地" → fetch + reset --hard
- "取消" → 跳过本次同步

对话框显示双方最后修改时间供参考。Git 历史保留所有版本，旧数据可随时回溯。

## 仓库认证

使用 **HTTPS + Personal Access Token** 方式认证。用户在设置页填入用户名和令牌，通过 HTTP Basic Auth 完成认证。

Token 存储在 OS 密钥链中（`uniTerm/git-token`），不在本地文件落盘。

## 仓库可见性

不做程序化检测，在 UI 中放置醒目的警告文字：

> **请使用私有仓库。** 配置文件包含加密后的密码和 API Key，为保证安全，请勿使用公开仓库进行同步。

## Git 实现

使用 `go-git` 纯 Go 库，不依赖系统安装 git。

## 敏感数据保护

### 主密码与加密密钥

用户在配置仓库时设置一个**主密码**。同一密码在不同机器上派生出相同的加密密钥，实现跨机器解密。

**密钥派生：** 主密码 + 随机 salt → PBKDF2（SHA-256, 600000 轮迭代）→ 32 字节 AES-256 密钥

**Salt：** 随机生成，存入远端仓库 `.sync-salt` 文件。新机器 clone 时获取 salt，用户输入相同主密码即可派生相同密钥。

**密钥缓存：** 派生后的密钥存入 OS 密钥链（`uniTerm/encryption-key`），日常使用无需重复输入主密码。

### OS 密钥链

使用 `github.com/zalando/go-keyring` 跨平台库：
- Windows：凭据管理器
- macOS：Keychain
- Linux：Secret Service / D-Bus

存储内容：
- `uniTerm/encryption-key`：派生后的 32 字节 AES-256 密钥
- `uniTerm/conn/<connection-id>`：连接密码
- `uniTerm/git-token`：Git PAT（仅 token 认证模式）

### 字段加密

推送前加密、拉取后解密。本地 JSON 文件保持明文。

**算法：** AES-256-GCM
- 随机 12 字节 nonce
- 密文 + nonce → Base64 编码

**加密字段：**
- `connections[].password`
- `ai-config.apiKey`

## 连接密码迁移

应用启动时执行一次存量迁移：

1. 加载 `connections.json`
2. 遍历每个连接，若 `authType == "password"` 且 `password` 非空：
   - 写入密钥链 `uniTerm/conn/<id>`
   - `password` 字段置空
3. 保存 `connections.json`

后续新增/修改连接时直接写密钥链。加载时从密钥链读取填充到内存。删除连接时清理密钥链对应条目。

密钥链不可用时回退读取 JSON 中 `password` 字段。迁移失败不阻塞主流程。

### 同步时的密码回填

`connections.json` 中的 password 字段已空（密码在密钥链），同步上传前需回填：

1. 读取 `connections.json`（password 为空）
2. 遍历每个连接，从密钥链读取密码
3. 回填到 JSON 结构中
4. 加密各字段 → 写入 repo → commit → push

拉取时反向操作：解密 `connections.json` → 如果 password 非空 → 写入密钥链 → JSON 中置空 → 保存。这样密码跨机器完整传输且本地无明文落盘。

## 模块架构

```
backend/
  sync/
    git.go           -- go-git 封装 (clone/pull/push/status)
    keychain.go      -- OS 密钥链封装
    crypto.go        -- AES-256-GCM 加密/解密
    sync_service.go  -- 同步编排 (手动/自动同步流程)
    sync_config.go   -- 本地同步配置持久化
```

远程仓库目录结构：
```
<repo>/
  .sync-salt          -- 密钥派生用随机盐
  connections.json    -- 加密后的连接配置
  ai-config.json      -- 加密后的 AI 配置
```

本地同步配置文件 `sync-config.json`（存于 `os.UserConfigDir()/uniTerm/`，不参与同步）：

```json
{
  "repoUrl": "https://github.com/user/config-repo.git",
  "branch": "main",
  "username": "your-username",
  "autoSync": true,
  "lastSyncAt": "2026-05-24T14:30:00Z",
  "lastSyncStatus": "success",
  "lastSyncError": ""
}
```

PAT 通过 OS 密钥链存储（`uniTerm/git-token`），不在 sync-config.json 中落盘。

### 交互说明

- **添加仓库**：弹窗填写仓库地址、用户名、令牌和主密码，保存后生成 salt、派生密钥、clone 仓库并执行首次同步
- **编辑仓库**：仓库地址只读锁定；可修改用户名、令牌；切换仓库需先删除再添加
- **修改仓库主密码**：同步卡片提供快捷入口，需验证当前密码并确认新密码。用新密钥重新加密所有配置文件并推送，其他设备需使用新密码
- **删除仓库**：二次确认后解除本地与仓库关联并清除本地凭证，本地配置和远端仓库不受影响
- **测试连接**：验证仓库可达且认证正确，在仓库配置卡片内操作
- **立即同步**：自动判断本地/远端差异并执行对应操作，有冲突时弹出冲突对话框
- **自动同步**：一行开关，开启后保存配置时自动推送、启动时自动拉取；关闭后纯手动
- **冲突取消**：跳过本次同步，两个版本保持不变，下次同步继续提示

## 错误处理

### 仓库连接

| 场景 | 提示 |
|------|------|
| 仓库地址不可达（网络/URL 错误） | "无法连接到仓库：<详情>" |
| Token 无效或已过期 | "认证失败：令牌无效或已过期" |
| 仓库不存在或无权限 | "仓库不存在或无权访问" |
| 远端仓库为空 | 自动初始化并推送（非错误） |

### 主密码

| 场景 | 提示 |
|------|------|
| 连接已有仓库时主密码错误 | "主密码错误，无法解密远端配置" |
| 修改仓库主密码时当前密码错误 | "当前密码错误" |
| 新主密码与确认不一致 | "两次输入的新密码不一致" |
| 添加仓库时主密码为空 | "请设置主密码" |

### 同步操作

| 场景 | 提示 |
|------|------|
| 未配置仓库时点立即同步 | "请先添加仓库配置" |
| 正在同步中再次点击 | "同步正在进行中，请稍后" |
| push 失败（远端有新提交） | "推送失败，远端有更新，请先拉取"（重试后自动变为冲突） |
| pull 失败（本地有未提交改动） | 不适用：流程是先 commit 再 pull |
| 本地仓库损坏 | "本地同步仓库异常，请删除仓库配置后重新添加" |
| 远端 `.sync-salt` 缺失 | "远端仓库数据异常，缺少密钥盐值" |
| 加密失败 | "加密配置文件失败：<详情>" |
| 解密失败 | "解密远端配置失败，主密码可能不匹配" |
| OS 密钥链不可用 | "系统密钥链不可用：<详情>"（回退读取 JSON 明文字段） |

### 显示方式

- **同步卡片**：始终显示最近一次同步时间和状态（成功/失败），失败时附带原因。手动和自动同步均更新此状态
- **手动同步（立即同步按钮）**：结果即时反映在同步卡片中，同时辅以 ElMessage toast
- **自动同步**：后台运行不弹窗，结果反映在同步卡片中；检测到冲突时弹出冲突对话框
- **测试连接**：ElMessage toast，成功绿色/失败红色
- **弹窗内错误**（添加仓库、编辑仓库、修改仓库主密码）：错误信息显示在弹窗底部，红色文字

```

                                ├── crypto.go (加解密)

                                │
                              本地文件 ↔ 远程 Git 仓库
```
